Modalità di esame

L'esame si compone di due parti complementari pensate per misurare competenze pratiche e teoriche.

  • Progetto: sviluppo di una soluzione cloud native completa.
  • Prova orale: discussione tecnica e domande sugli argomenti del corso.

Linee guida

  • Comprendi e documenta l'uso di librerie e servizi esterni.
  • Evita il riuso non dichiarato o l'uso inconsapevole di codice generato da AI.
  • Prepara la discussione motivando scelte architetturali e tecnologiche.

Progetto

Gli studenti possono lavorare individualmente o in gruppo (max 4 persone) per deliverare un'applicazione completa:

  • Frontend
  • Backend
  • Database

Tema e stack sono liberi. Il deployment può avvenire su provider cloud o in locale.

Pitch finale

Ogni squadra presenta il proprio lavoro con un pitch di 10 minuti: consigliati slide e demo live per mostrare la soluzione end-to-end.

Consegna del progetto

I materiali devono essere caricati su un repository pubblico (GitHub/GitLab) almeno 7 giorni prima dell'esame e notificati via email.

  • Codice sorgente e documentazione completa.
  • README con funzioni, architettura, istruzioni e credenziali di prova.
  • Diagrammi architetturali e di deployment.
  • File YAML di configurazione.
  • Pipeline CI/CD o script di build/test.
  • Link al deployment pubblico oppure istruzioni per l'avvio locale.

Best practice

Il README deve permettere un build from scratch senza supporto del team: cura chiarezza, referenze e automazioni.
